Create or update `docs/sdlc/current/02-requirements.md`.

Use `docs/sdlc/current/01-intake.md` when it exists. Include:

- Functional requirements with stable IDs, such as `FR-001`.
- Non-functional requirements with stable IDs, such as `NFR-001`.
- Acceptance criteria for each functional requirement.
- Out-of-scope items.
- Traceability table from requirement to planned verification.

If requirements are ambiguous, write the ambiguity explicitly and propose a conservative default.
