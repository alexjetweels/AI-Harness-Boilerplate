# Prompt: Review UI PR

```text
Review this UI change as a senior frontend engineer and product UI reviewer.

Check:
- Does it satisfy the requirement?
- Does it reuse existing patterns?
- Are tokens used instead of hardcoded values?
- Is the visual hierarchy clear?
- Does it avoid generic AI-looking UI?
- Are accessibility requirements met?
- Are hover/focus/disabled/loading/error states handled?
- Is it responsive?
- Are tests/stories updated?
- Is the code maintainable?

Return:
1. Overall score /10.
2. Must-fix issues.
3. Nice-to-have improvements.
4. Accessibility notes.
5. Token/design-system notes.
6. Suggested code changes if needed.
```
