# Anti-AI-Look UI Checklist

Use this checklist before shipping generated UI.

## Visual anti-patterns

Avoid:

- purple/blue gradient hero by default
- excessive glassmorphism
- glowing borders
- oversized rounded cards
- random decorative blobs
- too many icons
- generic feature cards
- same-height cards with fake content
- placeholder text as final UI
- center-aligning everything
- giant whitespace without hierarchy
- inconsistent spacing
- inconsistent border radius
- heavy shadows on every surface
- decorative motion that adds no meaning

## Better choices

Prefer:

- a clear information hierarchy
- realistic density
- purposeful surfaces
- token-based spacing
- subtle elevation
- consistent shape scale
- meaningful labels
- real data examples
- clear primary/secondary actions
- visible edge states
- responsive layout decisions
- accessibility-first interaction

## Product realism check

Ask:

1. Would a real user know what to do next?
2. Is there enough context to make a decision?
3. Are primary and secondary actions visually distinct?
4. Are errors recoverable?
5. Does empty state tell the user how to proceed?
6. Does loading state preserve layout stability?
7. Does the UI still work on mobile?
8. Does dark mode remain readable?
9. Would this screen look credible in production?
